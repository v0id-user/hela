# observability

Every deployable service exposes the same minimal surface:

| endpoint/header  | meaning                                                            |
| ---------------- | ------------------------------------------------------------------ |
| `GET /health`    | liveness check; JSON for Elixir services, `ok` for static apps     |
| `GET /version`   | service name, package version, commit, environment, generated time |
| `x-request-id`   | request correlation id generated by `Plug.RequestId`               |
| `x-hela-service` | `control` or `gateway` on Elixir APIs                              |
| `x-hela-version` | application version from the BEAM release                          |
| `x-hela-commit`  | deploy commit when the platform exposes it                         |
| `x-hela-region`  | gateway region slug                                                |

## control plane

Use `/health` for liveness and `/version` for deploy verification.
All logs include Plug request-id metadata when the logger formatter emits it.

Important operator env:

- `ALLOWED_ORIGINS`
- `HELA_SIGNUP_MODE`
- `HELA_INVITE_CODES`
- `POLAR_*`
- `GATEWAY_INTERNAL_SECRET`

## gateway

Use `/health` for liveness and `/version` for deploy verification.
`/cluster/snapshot` remains public because it powers the hosted status view.
`/_internal/cluster/reset` requires `x-hela-internal` and clears node runtime
state only; it does not delete Postgres history.

Gateway metrics currently cover:

- open channel count
- ingest/persist rates
- cache counts by project
- quota counters by project
- latency snapshots for broadcast and persistence
- BEAM process, memory, ETS, port, scheduler, and run queue data

## static apps

`web`, `app`, and `docs` emit `/version` from build-time `version.json`.
CI checks the deployed `/version.commit` against the pushed SHA after
Railway deploys.

## gaps

There is no OpenTelemetry exporter yet. For production-grade hosted use,
add OTLP traces/metrics, dashboard templates, log redaction checks, and
alert rules around:

- auth 429 rate
- gateway publish 429 rate
- Postgres persistence lag
- Broadway queue depth
- memory and ETS growth
- failed control-to-gateway sync
